## 二、5大区域速记表

| 区域 | 存储内容 | 线程 | 异常 | 调优参数 |
|------|---------|------|------|---------|
| **堆** | 对象实例、数组 | 共享 | OutOfMemoryError | -Xms -Xmx |
| **方法区/元空间** | 类信息、常量、静态变量 | 共享 | OutOfMemoryError | -XX:MetaspaceSize |
| **虚拟机栈** | 栈帧（局部变量、操作数栈） | 私有 | StackOverflowError | -Xss |
| **本地方法栈** | Native方法执行数据 | 私有 | StackOverflowError | -Xss |
| **程序计数器** | 字节码行号 | 私有 | **无** | 无 |

## 三、面试话术（背诵版）

### 开场白（30秒）
```
JVM内存结构主要分为线程共享区和线程私有区。

线程共享区包括堆和方法区，所有线程都可以访问；
线程私有区包括虚拟机栈、本地方法栈和程序计数器，每个线程独立拥有。

此外还有一个直接内存，是NIO使用的，不属于JVM运行时数据区。
```

### 堆（1分钟）
```
堆是JVM中最大的一块内存区域，被所有线程共享。

主要存放对象实例和数组，是垃圾回收的主要区域。
从GC角度，堆又分为新生代和老年代：
- 新生代：新创建的对象，分为Eden区和两个Survivor区
- 老年代：存活时间长的对象，大对象直接进入

JDK8后，字符串常量池也从方法区移到了堆中。
```

### 方法区/元空间（1分钟）
```
方法区存储已被加载的类信息、常量、静态变量、即时编译器编译后的代码等。

JDK8之前叫永久代（PermGen），使用JVM内存，容易OOM；
JDK8之后改为元空间（Metaspace），使用本地内存，默认只受物理内存限制。

可以通过-XX:MetaspaceSize和-XX:MaxMetaspaceSize来调整大小。
```

### 虚拟机栈（1分钟）
```
虚拟机栈是线程私有的，每个方法执行时都会创建一个栈帧。

栈帧包含：
1. 局部变量表：存放基本数据类型和对象引用
2. 操作数栈：方法执行时的运算空间
3. 动态链接：指向运行时常量池的方法引用
4. 方法返回地址：方法执行完后的返回位置

方法调用就是压栈，方法结束就是弹栈。
如果栈深度超过限制，会抛出StackOverflowError。
```

### 程序计数器（30秒）
```
程序计数器是线程私有的，记录当前线程执行的字节码行号。

作用是：
1. 字节码解释器通过它来获取下一条要执行的指令
2. 线程切换后能通过它恢复到正确的执行位置

它是唯一一个不会抛出OutOfMemoryError的区域。
```

### 本地方法栈（20秒）
```
本地方法栈和虚拟机栈类似，但它是为Native方法服务的。

在HotSpot虚拟机中，本地方法栈和虚拟机栈是合二为一的。
```

## 四、高频面试题

### Q1：堆和栈的区别？

| 对比项 | 堆 | 栈 |
|--------|-----|-----|
| 存储内容 | 对象实例、数组 | 栈帧（局部变量、操作数栈）|
| 线程 | 线程共享 | 线程私有 |
| 生命周期 | 对象创建到GC回收 | 方法调用到结束 |
| 异常 | OOM | StackOverflowError |
| 分配效率 | 慢（需考虑线程安全）| 快（栈顶分配）|

### Q2：为什么要把字符串常量池移到堆中？
```
永久代（方法区）空间有限，字符串常量池如果太大容易导致OOM。

移到堆中后：
1. 堆空间更大，减少OOM风险
2. 字符串生命周期更灵活，可以被GC回收
3. 与字符串对象存储一致，简化内存管理
```

### Q3：元空间和永久代的区别？
```
1. 存储位置：永久代在JVM内存，元空间在本地内存
2. 大小限制：永久代有固定上限，元空间默认只受物理内存限制
3. 垃圾回收：永久代FGC才回收，元空间更灵活
4. 性能：元空间减少FGC频率，提升性能
```

### Q4：什么情况下栈会溢出？
```
1. 无限递归调用（最常见）
2. 方法调用链过长
3. 局部变量过多，栈帧过大

解决：
- 检查递归终止条件
- 使用尾递归优化
- 增大栈空间（-Xss）
```
